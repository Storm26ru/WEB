WinSock

Windows Sockets API (WSA), название которого было укорочено до Winsock. Это техническая спецификация, которая определяет, как сетевое программное обеспечение Windows будет получать доступ к сетевым сервисам, в том числе TCP/IP. Он определяет стандартный интерфейс между клиентским приложением (таким как FTP-клиент или веб-браузер) и внешним стеком протоколов TCP/IP. Он основывается на API модели сокетов Беркли, использующейся в BSD для установки соединения между программами.

Дескрипторы (Тэги) физического форматирования

  • <b> (Boald) — жирный шрифт
  • <i> (Italic) — курсивный шрифт
  • <u> (Underline, Underscore) — подчеркнутый шрифт
  • <s> (Strikeout) — перечеркнутый шрифт
  • <t> (TT) — моноширный,моноширный шрифт, это шрифт у которого каждый символ занимает одинаковую площадь на экране, независимо от фактического размера символа

Дескрипторы (Тэги) логического форматирования

  • <strong> (Strong) — жирный шрифт;
  • <em> (Emphasize) — обозначить особым образом;
  • <strike> (Strike) — Перечеркнутый;
  • <code> (Code) — обозначить особым образом;
  • <pre> (Preformatted) — предварительное форматирование;
  • <span> () — этот дескриптор вообще ничего не делает, его используют для классификации элементов разметки;
